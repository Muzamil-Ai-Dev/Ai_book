# Data Model: Book Content

**Feature**: Book Platform Infrastructure **Date**: 2025-12-16

## File System Entities

Since this is a static site feature, the "Data Model" primarily concerns the file structure and
Frontmatter metadata of the Markdown files in `content/modules`.

### 1. Module (Chapter) File

**Path**: `content/modules/{slug}.md` or `content/modules/{slug}/index.md`

| Field              | Type    | Required | Description                                      |
| ------------------ | ------- | -------- | ------------------------------------------------ |
| `id`               | String  | Yes      | Unique identifier for Docusaurus routing         |
| `title`            | String  | Yes      | Display title of the chapter                     |
| `sidebar_label`    | String  | No       | Shorter title for sidebar navigation             |
| `sidebar_position` | Integer | Yes      | Order in the sidebar (1-8 for mandatory modules) |
| `description`      | String  | Yes      | SEO description and module summary               |
| `draft`            | Boolean | No       | If true, only visible in dev mode                |

### 2. Sidebar Configuration

**Path**: `apps/web/sidebars.js`

The sidebar will be **autogenerated** based on the file structure in `content/modules`, relying on
`sidebar_position` frontmatter.

## Relationships

- **Platform** (`apps/web`) -> **Reads** -> **Content** (`content/modules`)
- **Deployment** -> **Compiles** -> **Static Assets**
